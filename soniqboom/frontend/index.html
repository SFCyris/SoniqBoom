          📁 Music Folders

          🚦 FTP Connection Pools

          Tune how many concurrent FTP connections the scanner and the player can use per server. Stream borrows always get queue-jump priority — so playing a track during a heavy scan still feels instant. If the server allows fewer than your configured total, we automatically clamp to its limit.

          📡 Auto-Refresh (Remote Shares)

          FTP / SMB shares have no native push notifications, so the server polls them on an adaptive schedule (5 min for active shares, 4 h for stable ones). Click "Check now" to force a refresh. A toast appears only when new tracks are found — quiet otherwise.

          🌐 Find music — archives & demoscene

          🌐 Chiptune & tracker archives

          External links — each card opens an archive in a new tab. SoniqBoom does not auto-ingest from these. Download what you want, drop it in a scan directory, and the renderers (sidplayfp / libgme / openmpt) handle playback. HVSC metadata + Songlengths wire in via the HVSC panel in Settings.

          • 🌐 HVSC High Voltage SID Collection — the canonical Commodore 64 .sid archive (~60k tunes). Pair with Songlengths.md5 for per-tune durations. .sid / .psid
          • 🌐 The Mod Archive Largest tracker module collection on the web (~280k modules). Curated genres and Amiga/PC tracker scene compilations. .mod / .xm / .it / .s3m / …
          • 🌐 SNDH archive Atari ST / STe / Falcon YM2149 chiptunes. Use the libgme renderer (covers SNDH via .sndh / .ym). .sndh / .ym
          • 🌐 ZXAART (ZX-Art) ZX Spectrum AY-3-8910 chiptunes plus demoscene classics. AY files render via libgme; raw Spectrum modules via openmpt. .ay / .pt3
          • 🌐 AMP / AHX Amiga Music Preservation's AHX section. AHX is Amiga's 4-channel chiptune format; rendered via openmpt's AHX support. .ahx / .hvl
          • 🌐 Amiga Music Preservation Top-level AMP catalog: ProTracker / OctaMED / FastTracker / Composer modules from the Amiga demoscene. Browse by author or by year. .mod / .med / …

          Want a real remote-source ingest of one of these? Open an issue with the FTP / HTTP listing format and we'll plumb it through the FileSource backend.

          ✨ Demoscene — latest releases

          Where the newest scene music lands — party music-compo entries and fresh uploads. Same flow as above: download into a scan directory (scene.org also exposes FTP, so a remote FileSource can point straight at it).

          • 🌐 scene.org — parties The demoscene file archive. Each party folder (Revision, Assembly, The Gathering …) holds that event's music-compo entries — the freshest tracked & chip music, right after the show. FTP / open-directory access for bulk grabs. .mod / .xm / .it / .sid / …
          • 🌐 Demozoo The scene's release database. Filter to Music productions and sort by newest for direct downloads with proper composer / group credits. tracked & streamed music
          • 🌐 Pouët The scene's prod board — “latest released prods” including musicdisks, with votes & comments to surface the standouts. demoscene prods / musicdisks
